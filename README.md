# Прогнозирование отклика клиента на кредитное предложение

## Описание задачи

оценить вероятность согласия клиента на кредитное предложение.Модель должна оценивать, примет ли клиент конкретный оффер по кредитному продукту: бизнес-кредиту, возобновляемой кредитной линии или овердрафту.

**Метрика качества:** ROC AUC

## Подход

1. Feature Engineering: временные признаки, missing-флаги, interactions, target encoding
2. 7 моделей с 5-Fold Stratified CV на двух наборах признаков
3. Финальный ансамбль: простое среднее всех моделей
## Результаты

| Подход | OOF AUC | Комментарий |
|--------|---------|-------------|
| Лучшая индивидуальная модель (XGBoost v2) | 0.8359 | lr=0.01, depth=5 |
| **Простое среднее 7 моделей** | **0.8369** | **Финальное решение** |

### Особенности данных

**Сильный дисбаланс классов.** Только 6.09% клиентов соглашаются на предложение
(8 846 из 145 241). Соотношение классов примерно 16 к 1.

**Большое число пропусков.** В 20 из 27 признаков есть пропуски. В некоторых
столбцах пропущено до 96% значений. Пропуски не случайны — они означают
отсутствие у клиента определённого продукта или опыта (например, нет истории
овердрафтов, нет корпоративных кредитных продуктов).

| Признак | Доля пропусков | Смысл пропуска |

**Временной дрифт.** Доля положительных откликов существенно меняется со временем.
В ранних данных она составляет около 3%, в поздних — до 12.5%.

| Период (дециль) | Количество | Доля согласий |
|-----------------|------------|---------------|
| Начало (дни 0–41) | 15 189 | 3.07% |
| Середина (дни 155–192) | 14 075 | 6.03% |
| Конец (дни 274–390) | 14 578 | 12.51% |

**Тестовая выборка — продолжение обучающей по времени.** Обучающие данные
охватывают дни 0–490, тестовые — дни 490–669. Пересечение составляет
всего 1 день. Это означает, что модель должна обобщаться на будущие данные.

## Подход к решению

### Конструирование признаков

Исходные 27 столбцов расширены до 57 признаков за счёт четырёх групп
дополнительных переменных.

**Временные признаки (5 штук).** Из столбца с датой извлечены месяц,
день недели, день месяца, квартал и порядковый номер дня от начала наблюдений.
Порядковый номер дня оказался одним из наиболее значимых признаков,
поскольку улавливает общий временной тренд.

**Индикаторы пропусков (8 штук).** Для 7 столбцов с наибольшим количеством
пропусков созданы бинарные флаги (1 — значение отсутствует, 0 — присутствует).
Дополнительно создан признак с общим числом пропусков по строке. Индикатор
пропуска в столбце overdraft_app_term_max_360 показал корреляцию 0.21 с целевой
переменной — это третий по силе отдельный признак.

**Производные признаки (15 штук).** Созданы на основе предметной логики:
- Произведения (кредитная история × операции, события × время в системе)
- Отношения (сумма кредита / лимит овердрафта, ставка / ставка ЦБ)
- Разности (тренд дебетовых операций, разброс лимитов)

Наиболее сильным оказался признак loan_rev_x_cnt_cred (произведение давности
кредита на число кредитовых операций) с корреляцией 0.31.

**Целевое кодирование (до 4 штук).** Для категориальных и малокардинальных
признаков вычислена сглаженная средняя целевая переменная по группам.
Для предотвращения утечки данных использована схема с перекрёстной проверкой:
каждый пример кодируется статистиками, вычисленными без его участия.

### Обучение моделей

Обучены 7 моделей градиентного бустинга трёх видов с 5-кратной
стратифицированной перекрёстной проверкой.

**Базовые модели (стандартные параметры, lr=0.02):**

| Модель | Набор | Особенности | CV AUC |
|--------|-------|-------------|--------|
| LightGBM v1 | v1 (53 признака) | 63 листа, ранняя остановка 200 | 0.8330 |
| CatBoost v1 | v1 (53 признака) | Глубина 8, балансировка классов | 0.8342 |
| LightGBM v3 | v3 (57 признаков) | То же + целевое кодирование | 0.8339 |
| CatBoost v3 | v3 (57 признаков) | То же + целевое кодирование | 0.8339 |
| XGBoost v1 | v3 (57 признаков) | Глубина 7, 65% столбцов | 0.8344 |

**Консервативные модели (замедленные, lr=0.01):**

| Модель | Отличия от базовых | CV AUC |
|--------|-------------------|--------|
| LightGBM v4 | 31 лист, регуляризация 1.0, 50% столбцов, до 15 000 итераций | 0.8350 |
| XGBoost v2 | Глубина 5, регуляризация 1.0, 50% столбцов, до 15 000 итераций | **0.8359** |

Консервативные модели показали лучшие индивидуальные результаты. Меньшая
скорость обучения и сильная регуляризация не дают модели подстроиться
под шум в данных.

### Ансамблирование

Финальное предсказание — **простое среднее вероятностей от всех 7 моделей**.
Этот подход показал лучший результат (OOF AUC = 0.8369) по сравнению с:
- Взвешенным средним с оптимизированными весами (0.8367)
- Взвешенной тройкой лучших моделей (0.8367)

Причина: корреляция между предсказаниями моделей составляет 0.95–0.99.
При такой высокой согласованности оптимизация весов склонна к переобучению
на валидационных данных.

## Что было опробовано, но не улучшило результат

### Стекинг

Поверх предсказаний 7 моделей обучались мета-модели (логистическая регрессия,
LightGBM). Лучший результат стекинга — 0.8368, что не превышает простое среднее.
Причина: при корреляции предсказаний 0.95–0.99 мета-модель не находит
дополнительных закономерностей.

### Псевдоразметка

Уверенные предсказания на тестовой выборке (верхние и нижние 5%) добавлялись
как дополнительные обучающие примеры. Прирост минимальный (+0.0001 по OOF AUC),
на лидерборде без изменений.

### Временные стратегии валидации

| Стратегия | CV AUC | Проблема |
|-----------|--------|----------|
| Стратифицированная (используется) | 0.835 | — |
| Последовательное разбиение по времени | 0.805 | Сильный дрифт целевой переменной |
| Группировка по месяцам | 0.833 | Нестабильность между фолдами |
| Расширяющееся окно | 0.769 | Мало данных в ранних фолдах |

Несмотря на то что тестовая выборка является продолжением обучающей по времени,
стратифицированная перекрёстная проверка показала лучший результат
на лидерборде. Это связано с тем, что она использует больше данных
для обучения в каждом фолде.

### Обучение с временными весами

Экспоненциально возрастающие веса для более свежих наблюдений.
Идея: модель должна сильнее учитывать поздние данные, которые ближе
к тестовой выборке. На практике не улучшило результат на лидерборде.

### Модели на подгруппах признаков

Три модели на непересекающихся группах (финансовые, поведенческие,
категориально-временные). Снизило корреляцию между моделями
(с 0.95 до 0.85–0.90), но индивидуальное качество упало до 0.827,
что перевесило выигрыш от разнообразия.
